# VeriMind-MedAudit

基于多智能体协同与证据链溯源的儿科用药合规审计系统。

## 1. 项目简介

VeriMind-MedAudit 面向儿科用药这一高风险场景，构建“检索-推理-审计”多智能体协同架构，对药品剂量、禁忌症、适应症和证据来源进行自动核验，辅助医生或学习者发现潜在用药风险。

本系统不替代医生诊断，仅作为医学知识核验、用药风险提示和教学演示工具。

## 2. 核心痛点

- 儿科用药高度依赖年龄、体重、日龄等个体参数
- 通用大模型容易生成无依据剂量或错误禁忌信息
- 传统 RAG 只能“检索+回答”，缺少安全门控和拒答机制
- 医疗场景需要可溯源、可复核、可解释的证据链

## 3. 技术架构

系统由三个核心智能体组成：

1. Retrieval Agent：负责医学指南、药品说明书与处方集检索
2. Reasoning Agent：负责剂量、禁忌症和适应症初步分析
3. Audit Agent：负责证据一致性校验、Trust-Score 计算和风险拦截

核心信任评分：

T = α · (S_ret · W_authority) + β · S_faith

当证据不足、来源冲突或信任分低于阈值时，系统触发拒答或人工复核提示。

## 4. 重点演示模块 (Demo Showcase)

| 模块 | 演示方式展示 |
|---|---|
| **药品知识检索** | 输入药名/症状，返回指南原文 |
| **儿科剂量核验** | 输入年龄、体重、药品，检查剂量是否越界 |
| **证据链溯源** | 每个结论都显示来源、页码、原文片段 |
| **安全拒答门控** | 证据冲突或缺失时显示“请人工复核” |

## 5. 技术栈

- Backend：Python, FastAPI
- Retrieval：ChromaDB / 混合检索引擎
- LLM：LangGraph / LangChain，适配 GLM / DeepSeek 等 API
- Frontend：React / Ant Design (开发中)
- Data：公开医学指南、药品说明书、国家基本药物处方集

## 6. 项目边界声明

本项目仅用于科研、教学和辅助审计演示，不构成真实临床诊断或处方建议。所有输出结果均需由专业医务人员复核。
