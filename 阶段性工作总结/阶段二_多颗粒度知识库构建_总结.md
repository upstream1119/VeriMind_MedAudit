# 阶段性总结：多颗粒度知识库构建 (阶段二)

## 📌 完成时间
2026年4月11日

## 🎯 阶段目标
构建专业级医药知识检索系统，将权威的临床指南（如《中国儿科超药品说明书用药专家共识》等）进行结构化、向量化处理，建立具备上下文感知能力的 128/512/1024 Token 三粒度金字塔检索池。

## ✨ 具体完成工作

### 1. 语料分块与向量化 (Document Processing)
- **多粒度分块器**：利用 `langchain_text_splitters.RecursiveCharacterTextSplitter` 实现了严格的三级截断策略（128-短路精确匹配；512-机制解释提取；1024-复杂病案完整文段）。
- **Embedding 接入**：基于智谱大模型 `glm-4-text-embedding` 完成密集向量化嵌入计算，为每一种粒度建立了独立的数据切片集。

### 2. 向量数据库落盘 (ChromaDB)
- **多 Collection 架构**：在 `/data/chroma_db` 本地落库，建立了对应的三个命名空间 `med_chunks_128`、`med_chunks_512`、`med_chunks_1024`。
- **权重元数据锚定**：在入库时，为每一条药学知识片段绑定了权威度权重 `W_authority=0.7/0.9`，直接从数据源提供可用于信任度计算的先验锚点。

### 3. 多策略混合检索层 (Hybrid Retriever)
- **检索算法集成**：融合了 Chroma 的 Dense 向量余弦检索与基于 BM25 的稀疏词汇检索（针对罕见专业药名强化召回率）。
- **意图自适应查询**：设计了能够随着 Agent 传参意图（DETAIL / CONCEPT / CONTEXT）自动挂载到对应粒度 Collection 的底层方法。

## 🚀 联调结果
通过独立测试脚本 `test_retriever.py` 验证：
- 高效完成 36 个分块片段的嵌入与向量存储。
- 基于“左氧氟沙星剂量”或“早产儿布洛芬”等查询，测试成功触发 128 粒度并精确返回高度匹配结果及 `W_authority` 元数据。

## ⏭️ 下一步计划 (阶段 2.5 & 3)
具备了底层血液供应（知识库）后，下一阶段将进入系统的大脑区域：
- 实现包含数学门控公式的 Trust-Score 审计网络。
- 引入 LangGraph 将检索、路由和审计串联为多 Agent 通讯图。
